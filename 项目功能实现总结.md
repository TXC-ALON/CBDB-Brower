# CBDB Explorer 项目功能与实现总结

## 功能总结

### 1. 桌面应用与数据库连接
- 基于 Electron 桌面壳，启动时自动尝试加载项目根目录 `cbdb_20260221.sqlite3`。
- 支持手动选择 SQLite 文件；左侧栏显示连接状态、数据库路径、只读/可写状态。
- 数据库可写时自动创建常用索引（姓名、朝代、关系表、地址表等）以提升检索性能。

### 2. 人物搜索系统
- 支持按姓名/拼音、朝代、官职关键字、科举关键字组合检索。
- 支持繁简体关键词兼容（基于 `opencc-js`），并做别名扩展、姓+字组合匹配。
- 结果分页加载，含关键词高亮、朝代与生卒年展示、首个官职展示。
- 检索结果排序采用相关度优先：精确匹配、姓+字/别名匹配、模糊匹配等分层排序。

### 3. 人物详情面板
- 展示人物基础信息：姓名、拼音、朝代、生卒年、性别、姓/名、表字、号、族裔。
- 分块展示：字号别名、官职履历、家族关系、科举记录、社会关系。
- 各分块按时间/序列排序并限制返回数量，保证可读性与性能。

### 4. 关系图谱
- 支持两种视图：
  - 综合关系网络（家族+社会关系混合）
  - 家族家谱视图（核心+家族关系分层）
- 综合网络支持动态力导布局开关（关闭后静态布局，开启后可拖拽）。
- 家谱视图使用固定坐标布局并显示关系边标签。
- 支持双击节点跳转到该人物关系图谱；若无图谱则弹窗提示。
- 支持图谱访问历史（后退/前进按钮 + 鼠标侧键）。

### 5. 地理分布地图
- 支持两种模式：
  - 聚合模式：显示全局高密度地点分布
  - 人物轨迹模式：跟随当前人物并展示迁徙轨迹
- 跟随人物时，默认起止年份会按该人物生卒年自动填充（可手动覆盖）。
- 轨迹支持自动播放/手动步进（上一步、下一步、进度条）。
- 点击地图点位可打开详情弹窗，显示地点信息、访问次数、时间跨度与可能活动原因（官职/科举/社会关系推断）。

### 6. 统计分析
- 统计卡片：人物总数、官职记录、亲属关系、科举记录、社会关系。
- 图表：朝代分布 Top30、科举十年趋势、官职分布 Top20、家族网络规模 Top20。

### 7. 左侧人物历史（全局）
- 历史记录位于左侧栏，以气泡形式展示，最多 50 条。
- 仅记录“人物搜索”选中的人物（不记录图谱跳转行为）。
- 支持点击气泡快速回选人物。
- 支持右键气泡打开删除菜单并二次确认删除，支持一键清空。

## 实现方法

### 1. 架构分层
- **前端渲染层**：React + Vite + ECharts（`src/pages/*`）。
- **桌面桥接层**：Electron Preload 通过 `contextBridge` 暴露 `window.cbdbApi`。
- **数据访问层**：Electron Main 使用 `better-sqlite3` 直接查询 SQLite。

### 2. IPC 通信模型
- 前端统一通过 `src/services/api.js` 调用：
  - `db:get-status` / `db:pick-file`
  - `search:people`
  - `person:detail`
  - `graph:relations`
  - `geo:distribution`
  - `stats:overview`
  - `lookup:dynasties`
- 前端页面与主进程解耦，页面只关心数据结构，不关心 SQL 细节。

### 3. 搜索实现策略
- 关键词预处理：繁简转换、多变体扩展、别名扩展、复姓拆分。
- SQL 由 `createSearchWhere` 动态拼装，支持组合筛选。
- 相关度表达式由 `buildRelevanceExpressions` 构建，返回多级 rank 字段并排序。
- 查询采用 `COUNT + LIMIT/OFFSET` 分页，避免一次性加载超大数据。

### 4. 图谱实现策略
- 综合关系网络：
  - 静态模式使用预置极坐标分组布局。
  - 动态模式使用 ECharts force，并通过隐形 anchor 抑制核心节点漂移。
- 家谱视图：
  - 基于 kinship step 与关系词推断进行亲属分桶（祖辈/父辈/配偶/同辈/子孙）。
  - 使用 rail + elbow 辅助点绘制折线结构，减少线段杂乱。
- 交互：双击节点导航 + 图谱访问历史管理（含鼠标前进后退键）。

### 5. 地图实现策略
- 地图底图：本地 `china.geo.json`，离线渲染。
- 数据模式：
  - 聚合模式按地点聚合人物数量；
  - 人物模式按地点与时间序列返回轨迹点。
- 动画与详情：
  - 基于 `activeRouteIndex` 控制轨迹播放与进度。
  - 点位弹窗结合人物详情推断活动原因，增强解释性。

### 6. 状态与持久化
- React 组件状态管理 + `useMemo/useEffect` 派生数据。
- 人物历史使用 `localStorage` 持久化（`cbdb.person.history.v1`）。
- 异步加载统一采用取消标记或请求序号，避免竞态覆盖。

## 项目细节

### 1. 目录结构（核心）
- `electron/main.cjs`：数据库连接、SQL、IPC 注册。
- `electron/preload.cjs`：向前端暴露安全 API。
- `src/App.jsx`：全局布局、左侧菜单、人物历史、路由 tab。
- `src/pages/SearchPage.jsx`：人物检索。
- `src/components/PersonDetail.jsx`：人物详情。
- `src/pages/GraphPage.jsx`：关系图谱与家谱。
- `src/pages/MapPage.jsx`：地理分布与轨迹。
- `src/pages/StatsPage.jsx`：统计分析图表。
- `src/styles/main.css`：全局样式与各模块样式。

### 2. 关键依赖
- `better-sqlite3`：高性能本地 SQLite 查询。
- `opencc-js`：繁简转换。
- `echarts` + `echarts-for-react`：图谱、地图、统计图。
- `electron`：桌面化运行。

### 3. 关键数据表（读取侧）
- 人物主体：`BIOG_MAIN`
- 朝代：`DYNASTIES`
- 亲属：`KIN_DATA` + `KINSHIP_CODES`
- 社会关系：`ASSOC_DATA` + `ASSOC_CODES`
- 官职：`POSTED_TO_OFFICE_DATA` + `OFFICE_CODES`
- 科举：`ENTRY_DATA` + `ENTRY_CODES`
- 地理活动：`BIOG_ADDR_DATA` + `ADDR_CODES` + `BIOG_ADDR_CODES`
- 字号别名：`ALTNAME_DATA` + `ALTNAME_CODES`

### 4. 运行与构建
- 开发：`npm run dev`
  - Web 端口：`5174`
  - Electron 等待 Web 就绪后启动。
- 仅前端构建：`npm run build:web`
- 桌面打包：`npm run build`（输出到 `release/`）

### 5. 当前边界与注意事项
- 前端打包产物体积较大（ECharts 与地图数据导致 chunk 偏大，构建会有体积告警）。
- 家谱关系判定依赖 CBDB 原始关系编码与文本描述，个别复杂称谓仍可能需要规则细化。
- 轨迹与地点原因推断基于时间重叠规则，属于“解释性推断”，不是事件原文。
